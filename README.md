## 📖 Как использовать

### 1. Загрузка или генерация SBOM

#### **Вариант A: Загрузка готового SBOM файла**
- Нажмите кнопку **"Upload SBOM"** в правом верхнем углу
- **Один файл**: Перетащите или выберите файл → автоматическая загрузка
- **Несколько файлов**: Первый загружается сразу, остальные ждут слияния
- Нажмите **"Merge & Load All"** для объединения всех файлов

#### **🆕 Вариант B: Генерация SBOM из исходного кода**
- Нажмите кнопку **"Generate from Code"** в правом верхнем углу
- **Из отдельных файлов**: Загрузите package.json, pom.xml, requirements.txt и др.
- **Из архива**: Загрузите .zip или .tar.gz архив проекта
- **Автоопределение**: Система автоматически определит тип проекта
- **Настройки**: Укажите параметры генерации (dev-зависимости, опциональные пакеты и т.д.)
- Нажмите **"Generate SBOM"** для создания SBOM

### 2. Навигация по интерфейсу
- **Таблица компонентов** (центр) - список всех компонентов с сортировкой
- **Дерево зависимостей** (справа) - визуализация связей между компонентами
- **Панель деталей** (крайняя справа) - подробная информация о выбранном компоненте
- **Фильтры** (сверху) - быстрый поиск и фильтрация

### 3. Работа с таблицей компонентов
- **Сортировка**: Кликните по заголовку столбца для сортировки (↑ по возрастанию, ↓ по убыванию)
- **Ленивая загрузка**: Прокрутите вниз для загрузки дополнительных компонентов
- **Выбор компонента**: Кликните по строке для просмотра деталей
- **Автоскролл**: Выбранный компонент автоматически прокручивается в поле зрения

### 4. Исследование дерева зависимостей
- **Навигация**: Кликните по узлу для выбора компонента
- **Масштабирование**: Используйте кнопки + и - или колесико мыши
- **Панорамирование**: Перетаскивайте для перемещения по диаграмме
- **Полноэкранный режим**: Нажмите кнопку полноэкранного режима (клавиша Escape для выхода)
- **Режим матрицы**: Автоматически включается для проектов без зависимостей

### 5. Управление панелями
- **Сворачивание**: Нажмите стрелку в заголовке панели для сворачивания
- **Адаптивное пространство**: Панели автоматически расширяются при сворачивании других
- **Полноэкранный режим**: Дерево зависимостей может занимать весь экран

### 6. Фильтрация и поиск
- **Поиск**: Введите название компонента в поле поиска
- **Фильтры**: Используйте чекбоксы для фильтрации по типу, лицензии, риску
- **Комбинирование**: Фильтры работают совместно для точного поиска
- **Сброс**: Очистите фильтры для просмотра всех компонентов

### 7. Анализ безопасности
- **Цветовые индикаторы**: Зеленый (низкий риск), желтый (средний), красный (высокий)
- **CVE счетчики**: Количество известных уязвимостей для каждого компонента
- **Детальная информация**: Кликните по компоненту для просмотра подробностей
- **Навигация**: Используйте панель деталей для изучения зависимостей

## 🖥️ Backend Setup (для генерации SBOM) 🆕

### Требования
- Python 3.8+ или Node.js 18+
- Установленные менеджеры пакетов для целевых языков

### Запуск Backend API

```bash
# Клонируйте backend репозиторий (если отдельный)
git clone <backend-repository-url>
cd sbom-backend

# Установите зависимости
npm install  # или pip install -r requirements.txt

# Запустите сервер
npm start  # или python app.py

# Backend будет доступен на http://localhost:5000
```

### API Endpoints

- `POST /api/generator/generate-from-code` - Генерация SBOM из файлов
- `POST /api/generator/generate-from-archive` - Генерация SBOM из архива
- `GET /api/generator/project-types` - Список поддерживаемых типов проектов
- `POST /api/generator/detect-project-type` - Автоопределение типа проекта

### Конфигурация

Установите переменную окружения `VITE_API_BASE_URL` в `.env` файле:

```env
VITE_API_BASE_URL=http://localhost:5000/api
```

> **Примечание**: Без Backend API будут доступны только функции загрузки готовых SBOM файлов. Генерация из исходного кода требует запущенного backend сервера.

## ⚡ Производительность и технические характеристики

### 📊 **Метрики производительности**
- **Время парсинга**: ~50ms для 100 компонентов
- **Время рендеринга**: ~16ms (60 FPS)
- **Память**: ~2MB для 1000 компонентов
- **Размер бандла**: ~200KB (gzipped)

### 🚀 **Оптимизации**
- **Ленивая загрузка**: 20 элементов за раз для больших таблиц
- **useMemo**: Кэширование тяжелых вычислений сортировки
- **Intersection Observer**: Эффективная подгрузка данных
- **Виртуализация**: Оптимизированный рендеринг больших списков

### 📱 **Поддерживаемые браузеры**
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

## 🚧 Текущие ограничения

- Поддержка только JSON форматов SBOM
- Ограничение размера файла: 10MB
- CVE данные в демо-версии являются мокированными
- Отсутствует персистентность данных (данные теряются при перезагрузке)
